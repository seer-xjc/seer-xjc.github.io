    用户管理

    • Why?
    • how?

    Why? #

    为避免多成员使用服务器时文件混乱，每位成员在使用服务器时需要单独管理自已的用户空间，原则上所有非共享的文件、代码等数据都在自己的用户空间内存储使用。

    how? #

    第一次使用服务器时，请按顺序进行以下操作添加用户，如已有用户账号则跳过： #

    • 开机，按 clt+alt+T 键打开命令行终端

    • 更新系统的软件包列表并安装所有可用的更新（注意：使用sudo命令需要验证拥有sudo权限的用户密码验证，请确认已知密码）：

      sudo apt update && sudo apt upgrade -y

    • 添加用户（为便于区分，用户名建议为名字手写字母，下面命令中用户名以 xxx 为例）

      sudo useradd -m -s /bin/bash xxx

    • 给用户设置密码：

      sudo passwd xxx

      之后根据提示输入新密码、再确认密码，如有提示密码过短不用理会，可以正常设置。

      经过以上操作，新用户添加成功。

    • 将用户添加到sudo组（可选）

      sudo usermod -aG sudo xxx

    以下命令每次使用服务器都可能用到： #

    • 删除用户（将用户家目录和邮箱一起删除）

      sudo userdel -r test2

    • 切换到用户xxx：

      su - xxx

      不要省略中间的 - ，如果省略，切换用户后，终端还位于之前用户的文件系统中，但此时默认无权限访问其他用户文件，无法操作（读写）当前目录及文件，需要手动跳转到自己的用户的文件系统，而su - xxx可在切换用户后自动跳转到新用户的家目录。
